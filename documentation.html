Documentation

Add HStats to your Hytale mod in less than a minute. Stats are anonymous, and server owners can disable HStats any time.

Latest HStats file

Grab the HStats java file from GitHub to drag-and-drop into your mod source.

HStats.java Repository

Quick start

  1. Register your mod on the HStats site and copy your Mod UUID.
  2. Copy the HStats class file from GitHub into your mod source (package name can be changed).
  3. In your plugin setup, add:
new HStats("INSERT MOD UUID", "1.0.0");
Example setup method
@Override
protected void setup() {
    super.setup();
    new HStats("TEST-PLUGIN-UUID", "1.0.0");
}

Do not modify the HStats file (besides the package name). If you change it without permission, your account can be banned. It also defeats the point... the site displays stats on your mod, and messing with the data defeats the purpose.

Privacy + Opt Out

Stats sent to HStats are anonymous. We do not collect player names, IPs, or any personal info. Server owners can opt out any time by flipping a single flag.

What HStats sends
  • Player Count
  • Server OS Name + OS Version
  • Java Version
  • CPU Core Count
  • Server UUID + Mod UUID + Mod Version

These are used to generate the charts you see on the dashboard.

Set enabled=false in the server UUID file to disable metrics.

Server UUID file

Each server gets a unique UUID the first time HStats runs. It is stored in a small text file in the server root so it stays consistent across restarts. 

hstats-server-uuid.txt

HStats - Hytale Plugin Metrics (hstats.dev)
HStats is a simple metrics system for Hytale plugins. This file is here because one of your mods/plugins uses it,
please do not modify the UUID. HStats will apply little to no effect on your server and analytics are anonymous,
however you can still disable it.

enabled=true
<server-uuid-here>

To disable metrics on a server, set enabled=false in this file. HStats will stop sending.
